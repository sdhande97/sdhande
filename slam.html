Autonomous Navigation of a Robot

Gazebo Environment

Robot Designed in Solidworks

Mapping the Environment

Mapped Environment

Autonomous Navigation of a wheel robot

Localization with random occuring obstacles

Project Details

  • Category: ROS
  • Simulator/Visualizer:Gazebo/Rviz
  • Libraries: GMapping
  • Programming Language: PYTHON

Objective:To make robot autonomously navigate in environment using Lidar, and ROS navigation stack.

ROS Navigation Stack: This is a powerful tool for mobile robots to move from place to place reliably. The actual work of navigation stack is to produce a safe path for the robot to execute, by processing data from odometry, sensors, and environment map.

This is a problem of SLAM, which translates to instant localization and map construction. That means the robot will determine its position in an unknown environment while building a map, and finally output a map as shown on 4th slide. The process of SLAM map construction is to measure the distance information between the robot and the surrounding environment with depth sensor such as Lidar. Here I have used the most commonly used SLAM algorithm that is Gmapping which builds 2D map using data on laser sensor measurements and the data on the position of the mobile robot.

The node slam_gmapping reads data from the laser and transformations and then create the Occupancy Grid Map based on that data. In OGM, the occupancy is represented by the integer value in the range from 0 to 100. A value of 0 means completely unoccupied and 100 means fully occupied (e.g., wall), and a special value is -1, which means an unknown area. The localization and map building processes are separated here, where the localization is performed first and then map is built.

After building a map of the environment, the next thing we need to implement is the localization. The robot should localize itself on the generated map, and for this we implement the AMCL technique. It is a probabilistic localization system for a robot moving in 2D which uses a map, laser scans, and transform messages and outputs pose estimates. AMCL uses adaptive particle filter for localization, which uses map of our environment where the robot is set to initial(0,0) position. Now when we give a target location for the robot, the AMCL localizes the robot and then it is fed into a move_base planner for path planning and navigation. Now as the robot moves forward, we generate new samples and each sample stores a position and orientation data representing the robot's pose with some weightage to each sample. Sensor readings are incorporated by re-weighting these samples and normalizing the weights.

The filter is “adaptive” because it dynamically adjusts the number of particles in the filter: when the robot’s pose is highly uncertain, the number of particles is increased; when the robot’s pose is well determined, the number of particles is decreased. This enables the robot to make a trade-off between processing speed and localization accuracy.
