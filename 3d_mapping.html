3D Reconstruction and Mapping using Octomap

3D Mapped Environment

Autonomous Navigating Robot with Simultaneous 3D mapping

3D Mapping by Drone

Project Details

  • Category: ROS
  • Simulator/Visualizer: Gazebo/Rviz, Octovis
  • Libraries: Octomap
  • Programming Language: PYTHON

Objective:To make robot autonomously navigate and simultaneous 3D mapping of an environment.

This is a 3D Mapping Problem, which translates to 3D profiling of an area or object, such as using a pair of a stereo camera/depth camera. Ground-based robots are limited to 2D navigation due to their dynamics. However, since the flying robots like quadrotor can also adjust its vertical position, 3D navigation can be implemented. Navigation in 3D enables the quadrotor more maneuverability to explore its environment, the ability to get a much more complete understanding of the environment, and also navigate the environment in more complex paths. This is especially useful when it comes to obstacle avoidance.

The 3D mapping is done using Octomap which generate's volumetric 3D environment model. This mapping approach uses octrees and probabilistic occupancy estimation. Each node in an octree represents the space contained in a cubic volume, usually called a voxel. This volume is recursively subdivided into eight sub-volumes until a given minimum voxel size is reached. The minimum voxel size determines the resolution of the octree. The Probabilistic Occupancy Estimation is a procedure which estimates the marginal probabilities of presence of object at every location in an area of interest under a simple appearance model, given binary images corresponding to the result of a background-subtraction from different viewpoints. This explicitly represents occupied space, free and unknown areas. The advantage of using Octomap is that it allows for efficient and probabilistic updates of occupied and free space while keeping the memory consumption at minimum.
